Structured documentation, ILS thinking, and practical tooling

I’ve spent years in IT support, troubleshooting systems, working with identity management, and writing internal documentation in Confluence. That background shaped how I approach technical writing: documentation should be clear, structured, maintainable, and useful in real workflows. Today, I’m building on that foundation through S1000D migration projects, DMC/DMRL tooling, ILS study, and XML-based documentation workflows for complex technical systems.

HX-20 Legacy Maintenance Manual to S1000D

A realistic S1000D migration case study showing how a fictional helicopter maintenance procedure can be audited, split into data modules, assigned DMCs, structured with applicability for MK1/MK2 variants, and assembled through a publication module. Includes source DOCX, PDF preview, mini DMRL, simplified S1000D-style XML modules, operational test module, and ICN-style technical illustration.

A browser-based viewer for S1000D-style XML data modules. The tool parses local XML files and presents DMC breakdown, issue data, applicability, safety information, requirements, procedures, description content, and raw XML/JSON. Built as a static HTML/CSS/JavaScript project for GitHub Pages.

DMC Generator — S1000D Companion Tool

An interactive browser-based tool for building and validating S1000D Data Module Codes (DMC). Supports all DMC segments with inline validation, colour-coded output, and a DMRL (Data Module Requirements List) builder with CSV and XML export. Works as a focused companion to the S1000D Web Toolkit.
